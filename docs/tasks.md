# Tasks

## Problem
Large initiatives require visibility into what has been implemented and what remains.

## Suggested tracking categories

- `Architecture`
- `Domain checks`
- `Rendering`
- `Configuration`
- `Docs`
- `Tooling`

## Task structure
For each task include:
1. User-facing problem statement.
2. Acceptance criteria.
3. Files impacted.
4. Test/fixture updates required.
5. Documentation updates.

## Completion definition
- Feature behavior implemented.
- Contract/tests updated.
- Docs and crate readme references updated.
- No unchecked regressions in output shape.

## Maintenance tasks (periodic)
- Validate schema and fixture sync.
- Refresh explain coverage.
- Check for stale allow-lists.
- Review CI defaults and diff scope assumptions.
