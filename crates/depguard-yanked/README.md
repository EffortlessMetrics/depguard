# depguard-yanked

Offline yanked-version index parsing and querying for `deps.yanked_versions`.

This crate provides deterministic, IO-free parsing of yanked version indices. It enables depguard to detect yanked crate versions without network access.

## Purpose

The yanked crate:
- Parses multiple yanked index formats
- Provides exact version lookup (no semver matching)
- Remains completely offline and deterministic
- Supports the `deps.yanked_versions` check

## Key Features

### Multiple Input Formats

The crate supports three input formats for flexibility:

#### JSON Object Map

```json
{
    "serde": ["1.0.188", "1.0.189"],
    "tokio": ["1.37.0"]
}
```

#### JSON Array

```json
[
    {"crate": "serde", "version": "1.0.188"},
    {"crate": "serde", "version": "1.0.189"},
    {"crate": "tokio", "version": "1.37.0"}
]
```

#### Plain Text Lines

```
serde 1.0.188
serde@1.0.189
tokio 1.37.0
# comments are ignored
```

### Exact Version Matching

The index uses exact version matching (not semver):

```rust
use depguard_yanked::{parse_yanked_index, YankedIndex};

let index = parse_yanked_index(r#"{"serde": ["1.0.188"]}"#)?;

assert!(index.is_yanked("serde", "1.0.188"));  // Exact match
assert!(!index.is_yanked("serde", "1.0.189"));  // Different version
assert!(!index.is_yanked("serde", "1.0"));      // Not exact
```

## Public API

```rust
/// In-memory yanked-version index
pub struct YankedIndex {
    // Internal BTreeMap storage
}

impl YankedIndex {
    /// Check if a specific version is yanked
    pub fn is_yanked(&self, crate_name: &str, version: &str) -> bool;
    
    /// Number of crates in the index
    pub fn len(&self) -> usize;
    
    /// Check if index is empty
    pub fn is_empty(&self) -> bool;
    
    /// Add a yanked version entry
    pub fn insert(&mut self, crate_name: &str, version: &str);
    
    /// Merge another index into this one
    pub fn merge(&mut self, other: YankedIndex);
}

/// Parse a yanked index from text (auto-detects format)
pub fn parse_yanked_index(input: &str) -> anyhow::Result<YankedIndex>;
```

## Usage Example

```rust
use depguard_yanked::{parse_yanked_index, YankedIndex};

// Parse from JSON
let json = r#"{"serde": ["1.0.188", "1.0.189"], "tokio": ["1.37.0"]}"#;
let index = parse_yanked_index(json)?;

// Query the index
assert!(index.is_yanked("serde", "1.0.188"));
assert!(index.is_yanked("serde", "1.0.189"));
assert!(!index.is_yanked("serde", "1.0.190"));

// Parse from plain text
let text = r#"
serde 1.0.188
tokio@1.37.0
# this is a comment
"#;
let index = parse_yanked_index(text)?;

// Merge indices
let mut main_index = parse_yanked_index(r#"{"serde": ["1.0.188"]}"#)?;
let extra_index = parse_yanked_index(r#"{"tokio": ["1.37.0"]}"#)?;
main_index.merge(extra_index);

assert_eq!(main_index.len(), 2);
```

## Integration with depguard

The yanked index is typically:
1. Generated by an external process (e.g., fetching from crates.io)
2. Stored as a file in the repository or CI environment
3. Loaded by depguard at runtime
4. Passed to the domain layer for check evaluation

```rust
// In depguard-app
let yanked_text = std::fs::read_to_string("yanked-index.txt")?;
let yanked_index = parse_yanked_index(&yanked_text)?;

// Pass to check evaluation
let report = evaluate(&model, &config, Some(&yanked_index));
```

## Design Constraints

- **No filesystem access**: Input is provided as string
- **No network access**: Completely offline
- **Deterministic**: Same input always produces same output
- **Exact matching**: No semver interpretation

## Generating a Yanked Index

The yanked index is typically generated externally. Example approaches:

1. **From crates.io API**: Query the API for yanked versions
2. **From local cache**: Parse Cargo's local index
3. **Manual maintenance**: Curated list for specific needs

The format is simple enough to generate with basic scripting:

```bash
# Example: Generate from crates.io API (pseudo-code)
curl -s https://crates.io/api/v1/crates/serde/versions | \
    jq -r '.versions[] | select(.yanked) | "serde \(.num)"' > yanked.txt
```
